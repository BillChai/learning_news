## Kotlin / Android / Swift

1. **[Kotlin] Kotlin 2.4.0 正式發布**
   context parameters 與 explicit backing fields 轉為 stable，Kotlin/JVM 支援 Java 26，Kotlin/Native 可將 Swift package 當作依賴、CMS GC 預設開啟。對後端與 KMP 開發者都是重要升級。
   來源：[The Kotlin Blog](https://blog.jetbrains.com/kotlin/2026/06/kotlin-2-4-0-released/)｜發布：2026-06（近期）

2. **[Android] JetBrains Koog + Android CLI 轉 stable，讓 AI agent 深入行動開發**
   Koog 是 JetBrains 的 Kotlin-native AI agent 框架，可本地運行、串接工具；同時 Android CLI 進入 stable，提供讓 AI agent 執行核心 Android 任務的程式化介面。值得關注 agent 化開發工具鏈的成熟度。
   來源：[AndroidDev News / Kotlin Roundup](https://blog.jetbrains.com/kotlin/2026/06/kodees-kotlin-roundup-kotlin-turns-15-kotlin-2-4-0-and-the-kotlin-toolchain/)｜發布：2026-06～07

3. **[Swift] What's new in Swift（June 2026 Edition）：TrueType 直譯器由 C 改寫為 Swift 且快 13%**
   Apple 將 TrueType Hinting Interpreter 由 C 重寫為 memory-safe Swift，效能反而比原 C 版快 13%，是「安全語言不必犧牲效能」的實證案例。文中也整理了 WWDC 後的 Swift 社群動態。
   來源：[Swift.org Blog](https://www.swift.org/blog/whats-new-in-swift-june-2026/)｜發布：2026-07-02

4. **[Swift] Swift Package Index 加入 Apple，並將打造正式 package registry**
   Swift Package Index 團隊加入 Apple 但維持開源，將與 Apple 合作建立完整的 Swift package registry；同時 Swift 生態新增 Networking workgroup 統一網路堆疊。對 Swift 依賴管理生態影響深遠。
   來源：[Swift.org Blog](https://www.swift.org/blog/whats-new-in-swift-june-2026/)｜發布：2026-07-02

## AWS / IaC

5. **[AWS] Graviton5 C9g 實例、EKS 版本回滾、CloudFormation Express mode**
   早 7 月一波更新：EC2 C9g/C9gd（Graviton5）較 Graviton4 效能提升達 25%；EKS 支援七天內回滾 Kubernetes 版本升級；CloudFormation Express mode 讓部署秒級確認，全商用區免費。直接影響成本與部署可靠度。
   來源：[AWS What's New](https://aws.amazon.com/new/)｜發布：2026-07（近期）

6. **[AWS] Claude Sonnet 5 上線 Bedrock，OpenAI GPT-5.5/5.4 進入 Bedrock 有限預覽**
   AWS Weekly Roundup 宣布 Claude Sonnet 5 於 Bedrock 可用，並與 OpenAI 擴大合作，將 GPT-5.5/5.4、Codex 及 Bedrock Managed Agents 帶入 Bedrock。多模型同平台對 LLM 應用架構選型很關鍵。
   來源：[AWS News Blog](https://aws.amazon.com/blogs/aws/aws-weekly-roundup-claude-sonnet-5-on-aws-amazon-workspaces-for-ai-agents-aws-service-availability-updates-and-more-july-6-2026/)｜發布：2026-07-06

7. **[AWS CDK] CDK Mixins GA + CloudFormation 部署前驗證**
   CDK Mixins（2026-03 GA）讓你對任何 construct（L1/L2/自訂）加上可組合、可重用的抽象，不必重寫既有 IaC；CloudFormation 現於 Create/Update Stack 前執行預驗證，秒級回報常見錯誤。最新 CDK 版本 2.1130.0（2026-07-02）。
   來源：[AWS CDK Mixins GA](https://aws.amazon.com/about-aws/whats-new/2026/03/aws-cdk-mixins/)｜發布：2026-03（CDK 每週更新）

   xAI 於 2026-07-08 發布 Grok 4.5；Anthropic 推出 Claude Fable 5（隨後因出口管制指令暫停存取）；Claude Sonnet 5 以 $2/$10（每百萬 token）成為 Free/Pro 預設，主打「接近前沿的 agentic 能力＋可負擔價格」。選型與成本模型都要重算。
   來源：[LLM Stats – AI News](https://llm-stats.com/ai-news)｜發布：2026-07-08

9. **[AI] 微軟推出 MDASH：多模型 agentic 安全平台自動化程式碼稽核**
   MDASH 是微軟的 AI 驅動漏洞發現系統，用多模型 agent 大規模自動稽核 Windows 等軟體程式碼。是「用 LLM agent 做防禦性安全」的代表性落地案例。
   來源：[LLM Stats – AI News](https://llm-stats.com/ai-news)｜發布：2026-07（近期）

10. **[AI] Meta 與 Nebius 簽 270 億美元、五年 AI 算力合約**
    含 120 億美元專屬基礎設施（早期大規模部署 Nvidia Vera Rubin 平台）＋ 150 億美元追加產能。反映前沿模型競賽下算力綁約的規模與趨勢。
    來源：[Large Language Models News July 2026](https://blog.mean.ceo/large-language-model-news-july-2026/)｜發布：2026-07（近期）

11. **[工程] Simon Willison：實作任務交給「低算力 subagent」，主 loop 留給判斷與整合**
    Willison 建議「所有 coding 任務用判斷力選一個較低算力模型、丟到 subagent 跑」，理由是實作工作很少需要頂級模型，而 review/synthesis 留在主循環。是實務省成本的 agent 工作流心法。
    來源：[Simon Willison's Weblog](https://simonwillison.net/)｜發布：2026-07-03

## 工程實務 / 架構 / 綜合

15. **[架構] Agentic coding 最佳實務：程式碼健康度是前提，並邁向 ASDLC**
    多方（InfoQ、CodeScene、Anthropic 2026 Agentic Coding Trends Report）共識：agent 化開發需要「更多而非更少」的結構與程式碼品質，低 Code Health 會讓 agent 更容易失敗或燒 token；並主張以 ASDLC（agentic SDLC）強調「agent 絕不可做什麼」，工程師仍是最終審核關卡。呼應 HN 近期「信任與安全 > 炫技」的趨勢。
    來源：[InfoQ – Prompts to Production](https://www.infoq.com/articles/prompts-to-production-playbook-for-agentic-development/)｜發布：2026（近期）
